# 智能拜访副驾 产品指南

## 目标
这是一个面向客户的产品版本，旨在展示：
- 拜访前医生准备和作战卡能力
- 拜访中实时副驾建议与风险提醒
- 会后自动生成纪要并回填 CRM
- 手机端低干扰体验与移动联动

## 运行方式
1. 进入项目目录：
   ```bash
   cd /Users/liuankun/ComateProjects/comate-zulu-demo-1779093923451/pharma-ai-mvp
   ```
2. 启动开发服务器：
   ```bash
   npm run dev -- --host 0.0.0.0
   ```
3. 访问产品页面：
   - 桌面浏览器：`http://localhost:5173/`
   - 手机体验：`http://172.26.112.156:5173/`

## PWA（移动 Web 小程序）说明

- 我们已添加基础 PWA 支持（manifest + service worker），用户可在手机浏览器通过“添加到主屏幕”安装为类小程序体验。
- 开发服务器启动后，打开产品首页并按浏览器提示安装；生产部署建议使用 Vercel / Netlify 或自有域名并启用 HTTPS。

示例部署（Vercel）：将 `pharma-ai-mvp` 目录用 Git 提交后在 Vercel 创建项目并选择 `npm run build` 作为构建命令，输出目录为 `dist`。

## 使用路径建议
1. 登录进入系统
2. 先进入“今日拜访助手”，展示医生画像、目标和作战卡
3. 再进入“拜访中副驾”，展示实时摘要与关键建议
4. 进入“会后纪要”，展示AI抽取结果与CRM回填
5. 最后进入“手机演示”，展示客户手机端视觉与低干扰提示

## 关键演示亮点
- 结构化拜访准备页面，让客户快速理解“医药代表能做什么”
- 实时建议页面，让客户看到“AI 如何辅助现场沟通”
- 会后纪要页面，让客户感受到“CRM 自动落地”的价值
- 手机演示页面，体现产品对移动端用户体验的重视
